ПОДКЛЮЧЕНИЕ ИКОНОК НА САЙТЕ

  1. Подключить иконку для вкладок браузера в формате svg

    <link rel="icon" href="img/icons/favicon.svg" sizes="any" type="image/svg+xml">

    или в формате png размерром 16x16 и 32x32 - для ретины

    <link rel="icon" href="img/icons/favicon-16x16.png" sizes="16x16" type="image/png">
    <link rel="icon" href="img/icons/favicon-32x32.png" sizes="32x32" type="image/png">

  2. Подключить иконку apple-touch-icon

    <link rel="apple-touch-icon" sizes="180x180" href="img/icons/apple-touch-icon-2-180x180.png" type="image/png">

  3. Подключить файл manifest.webmanifest или manifest.json

    <link rel="manifest" href="manifest.webmanifest">

    затем в файле manifest.webmanifest подключить тач-иконки для Андроид и все остальное:

    {
    //представляет имя веб-приложения, как оно обычно отображается пользователю
    "name": "New Project by Dots",

    //короткое имя веб-приложения - определяет название, показываемое пользователю
    "short_name": "NPD",

    "icons": [

    //массив объектов изображений, которые могут использованы как иконки приложения в различных контекстах
        {
        "src": "img/icons/touch-icon-192x192.png",
        "sizes": "192x192",
        "type": "image/png"
        }
    ],

    //предпочитаемый разработчиком режим отображения для веб-сайта
    //варьируется от "browser" (когда отображается полное окно браузера) до "fullscreen" (когда приложение полноэкранно).
    "display": "fullscreen",

    //ориентация экрана: any, natural, landscape, landscape-primary, landscape-secondary, portrait, portrait-primary, portrait-secondary
    "orientation": "landscape",

    //цвет темы - в шапке приложения
    "theme_color": "#00ff00",

    //цвет фона - определяет ожидаемый цвет фона для веб-приложения
    "background_color": "aqua",

    //область навигации контекста веб-приложения. Обычно ограничивается тем, какая веб-страница может быть просмотрена, пока манифест загружается. Если пользователь выходит за пределы области, приложение возвращается к нормальному виду веб-страницы.
    "scope": "./",

    //начальный URL-адрес веб-приложения — предпочтительный URL-адрес, который должен быть загружен при запуске пользователем веб-приложения (например, когда пользователь нажимает на значок веб-приложения в меню приложений или на домашнем экране).
    "start_url": "index.html"
}
